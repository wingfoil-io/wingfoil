# Plan: Proc-macro derives for StreamPeekRef and Upstreams (Issue #124)

## Goal

Eliminate boilerplate in custom node definitions by introducing a `wingfoil-derive` proc-macro
crate that generates `StreamPeekRef` and `WiringPoint` impls from field attributes.

## New User-Facing API

```rust
#[derive(new, StreamPeekRef, Upstreams)]
pub struct MapStream<IN, OUT: Element> {
    #[active]
    upstream: Rc<dyn Stream<IN>>,
    #[new(default)]
    #[output]
    value: OUT,
    func: Box<dyn Fn(IN) -> OUT>,
}

impl<IN, OUT: Element> MutableNode for MapStream<IN, OUT> {
    fn cycle(&mut self, _state: &mut GraphState) -> anyhow::Result<bool> {
        self.value = (self.func)(self.upstream.peek_value());
        Ok(true)
    }
    // no upstreams() needed — generated by #[derive(Upstreams)]
}
```

## Completed Work

### `wingfoil-derive` crate — `#[derive(StreamPeekRef)]`

- Generates `impl StreamPeekRef<T>` from the field tagged `#[output]`
- Compile error if zero or more than one `#[output]` field is present
- Re-exported as `pub use wingfoil_derive::StreamPeekRef` from `wingfoil`

### `#[derive(Upstreams)]` — `WiringPoint` supertrait pattern

**Rationale:** Rust only allows one `impl Trait for Type` block. Extracting `upstreams()` into
`WiringPoint` (a supertrait of `MutableNode`) lets the derive generate a separate
`impl WiringPoint` without conflicting with the user's `impl MutableNode`.

**New traits in `wingfoil/src/types.rs`:**

- `WiringPoint` — `fn upstreams(&self) -> UpStreams;`
  - Supertrait of `MutableNode`
  - `RefCell<NODE>` blanket impl delegates to inner
- `AsUpstreamNodes` — uniform helper for fields of type `Rc<dyn Node>`,
  `Rc<dyn Stream<T>>`, or `Vec<U: AsUpstreamNodes>`
  - Generated code calls `AsUpstreamNodes::as_upstream_nodes(&self.field)`
  - `Rc<dyn Stream<T>>` impl calls `as_node()` via the vtable (no `T: 'static` needed)

**`#[derive(Upstreams)]` attributes:**
- `#[active]` — upstream field triggers downstream cycles when it ticks
- `#[passive]` — upstream field is read but does not trigger
- No tagged fields → generates `UpStreams::none()` (source nodes)

**Migration pattern:**

| Category | Approach |
|----------|----------|
| Simple `Rc<dyn Stream<T>>` / `Rc<dyn Node>` / `Vec<...>` fields | Path A: `#[derive(Upstreams)]` + `#[active]`/`#[passive]` |
| `Dep<T>` fields (runtime active/passive) | Path B: manual `impl WiringPoint` |
| `Option<Rc<dyn Node>>` trigger fields | Path B: manual `impl WiringPoint` |
| Complex late-binding (demux, graph_node, channel) | Path B: manual `impl WiringPoint` |

All nodes in `wingfoil/src/nodes/`, `wingfoil/src/adapters/`, `wingfoil/src/bencher.rs`,
and `wingfoil-python/src/proxy_stream.rs` have been migrated.
